# West Monroe Brand Source

Use the West Monroe UX/UI guidelines repository as the source of truth when creating branded quickstart pages:

`https://github.com/jsargeant-dev/west-monroe-ux-ui-guidelines.git`

## How To Use The Repository

When working on a page, inspect the repository before creating custom brand patterns. Look first for:

- Brand guidelines
- Color and typography guidance
- Logo and image assets
- Navigation templates
- Hero templates
- Footer templates
- CTA sections
- Cards, stats, accordions, and common content sections
- Existing HTML/CSS examples that can be adapted

Reuse priority:

1. Navigation
2. Hero
3. Footer
4. CTA blocks
5. Common content sections
6. Supporting UI components

## Brand Defaults

Use West Monroe branding unless the user explicitly asks for client branding, co-branding, or a neutral wireframe.

When the page is for a client or proposal, still default to West Monroe branding and incorporate client-specific messaging through copy and content, not by changing the whole visual system.

## If The Repository Is Not Available

If the repo cannot be cloned or inspected, proceed with a conservative West Monroe-style consulting page:

- Clean, confident, business-focused layout
- Clear hierarchy and concise sections
- Practical, advisory voice
- Restrained color use
- Strong CTAs without aggressive sales language
- No generic startup/SaaS hero treatment

Note in the handoff that exact brand assets/templates were not verified.
